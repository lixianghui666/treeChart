class Bus{  constructor(){    this.cb = {};  }  on(evName,cb,once = false){    if(!this.cb[evName]){      this.cb[evName] = [];    }    this.cb[evName].push({      cb,once    });  }  once(evName,cb){    this.on(evName,cb);  }  emit(evName,cb,params){    let Listener = this.cb[evName] ? this.cb[evName] : [];    let result;    if(cb){      Listener.forEach((item,index) => {        if(item.cb === cb){          result = item.cb(params);          if(item.once){            Listener.splice(index,1);          }          return result;        }      });    }else{        result = [];        Listener.fotEach((item,index) => {          result.push(item.cb(params));          if(item.once){            Listener.splice(index,1);          }        });        return result;     }  off(evName,cb){    let Listener = this.cb[evName] ? this.cb[evName] : [];    if(evName){      if(cb){        Listener.forEach((item,index) => {          if(item.cb === cb){            Listener.splice(index,1);          }        });      }else{        delete this.cb[evName];      }    }else{      this.cb = {};    }  }}

function debounce(fn,time){  let lastTime,result;  return () => {    let now = new Date().getTime();    if(!lastTime || (now - lastTime) >= time){      result = fn();    }    lastTime = now;    return result;  }}

var dayOfTheWeek = function(day, month, year) {    const week = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"];    const monthDays = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30];    //输入年份之前的年份的天数贡献    let days = 365 * (year - 1971) + Math.floor((year - 1969) / 4);    //输入年份中，输入月份之前的月份的天数贡献    for(let i = 0; i < month - 1; ++i) {        days += monthDays[i];    }    if((year % 400 === 0 || (year % 4 === 0 && year % 100 !== 0)) && month >= 3) {        days += 1;    }    //输入月份中的天数贡献    days += day;    return week[(days + 3) % 7];};

async和defer区别

1.首先async和defer都不阻塞DOM渲染,正常script都会阻塞DOM渲染的
2. async 和defer都是异步加载的
3. 如果有多个defer,那么会按照顺序执行
4. 如果有多个async,谁先加载完谁先执行
5. defer会在DOMContentLoaded事件触发前执行完
6. async并不影响DOMContentLoaded事件执行

DOMContentLoaded和load区别

1. DOMContentLoaded事件会在HTML加载和解析完成之后就会执行,而不用等待样式文件和一些资源文件加载完成后才执行
2. load事件是需要全局的所有资源加载和解析完成之后才会执行

function add(num1: string,num2: string){  const maxLen: number = Math.max(num1.length,num2.length);  num1 = num1.padStart(maxLen,"0");  num2 = num2.padStart(maxLen,"0");  let f: number = 0,t: number = 0,sum: string = "";  for(let i = maxLen - 1;i >= 0;i--){    t = num1[i] + num2[i] + f;    f = ~~(t / 10);    sum = "" + (t % 10) + sum;  }  if(f != 0){    sum = "" + f + sum;  }  return sum;}

//普通的阶乘可能比较简单,使用递归或者循环就可以实现,但是如果要考虑到数值较大导致溢出就麻烦了function factorial(n){  if(typeof n !== "number")return throw TypeError(`${n} is not a numeric type`);  let result = [1],  digit = 1,  curry = 0,  num,  i,  count;  for(num = 2,curry = 0;num <= n;++num){    for(i = 0;i < digit;++i){      count = result[i] * num + curry;      result[i] = count % 10;      curry = (count - result[i]) / 10;    }    while(curry){      result[digit++] = curry % 10;      curry -= curry % 10;      curry /= 10;    }  }  return result.reverse().join("");}

cookie

cookie一般使用于存储用户状态信息数据的，它是以文件形式储存在浏览器中，它含有一些属性，设置字段有效期、安全性等，具体含义如下:

name这个字段用于设置cookie的某一个属性名,这个是动态的
value这个字段是设置属性值的,也是动态的
domain这个是规定哪个域下才能访问该cookie字段
path规定域下哪个路径才能访问该字段
expires这个是一个秒级的时间戳,该cookie字段会在该字段表示的时间之后删除字段
max-age这个是秒单位,过多久后过期,优先级高于expires,当属性值为0时会立刻删除,为负数时关闭浏览器会删除
secure设置cookie只能实在ssl和https下才会请求携带cookie
httpOnly设置这个后前端就不能通过document.cookie访问到cookie了，只能请求的时候携带到服务器上
sameSite设置跨域是否能够携带cookie,strict是表示任何跨域请求都不携带,lax只有a、link和get请求带上cookie,none都带上

跨域携带cookie还需要在请求头上设置withCredentials属性值为true

浏览器面试题

缓存

浏览器缓存分为强制缓存以及协商缓存两种

强制缓存

强制缓存依靠的是expires和catch-control这个两个头里面的字段值来判断是否采用缓存

1. expires: 这字段设置就是数据过期时间,如果当前时间大于expires的时间则向服务器请求数据,反之就是直接在浏览器中拿缓存,http1.0的产物,后面逐步被cache-control替代

2. cache-control: cache-control主要是根据它里面的字段值来判断是否可以缓存以及缓存的最大时长,这个的优先级要大于expires，具体的字段值如下:

private客户端可以缓存
public客户端和代理服务器都可以缓存
max-age数据过期时间
no-cache不管浏览器是否含有缓存都要请求服务器,由服务器确定
no-store请求报文或者响应报文中含有机密信息，不要缓存
协商缓存

1. last-modified与if-modified-since

首先浏览器在第一次请求服务器时，服务器会给浏览器的响应报文中带一个last-modified的字段,这个字段表示该文件最近一次修改的时间

下一次请求服务器的时候就会带一个if-modified-since字段在请求报文里面,值是上一次服务器返回的last-modified的值,服务器获取到该字段值与文件的最新一次修改的时间作对比,如果最近修改的时间小于等于if-modified-since的值,就返回304,反之返回200,并且再次更新返回last-modified的值，将最新的文件发送给客户端

2. etag与if-no-match

由于上面那个协商缓存只能精确至秒单位,而有些功能需要毫秒级响应,所以可能就不匹配需求,或者一个资源被修改了，但其实际内容根本没发生改变，服务器还是返回了数据

首先浏览器在第一次请求服务器时会带一个etag,这个字段值是一个特殊的字符串,且各不相同,服务器也会将其存储起来,下一次浏览器再次请求服务器的时候会携带一个if-none-match的字段,值就是上次请求etag字段的值,服务器会比较两者是否相同,如果一样则返回304,如果不一样则将更新etag，返回数据和200

缓存好处:

1. 减少冗余数据传递

2. 减少服务器请求压力,提高网站性能

3. 加快了网页的加载速度

浏览器输入ulr过程

1. 首先进行DNS解析，将域名解析成ip地址

2 . 根据ip找到相应服务器进行tcp连接(三次握手)

3. tcp连接成功后客户端向服务器发起http请求

4. 服务器接收到客户端的请求处理再将数据响应客户端

5. 客户端接收到服务器返回的数据,解析数据开始渲染页面(解析html树形成dom树,解析css形成cssom树,合并形成渲染树)

6. 关闭tcp连接(4次挥手)

重绘和回流

回流: 页面上的某些元素的位置或者结构发生改变,导致浏览器要部分或者全部渲染的过程称之为回流

重绘: 当一个元素外观发生改变时就会导致重绘

导致重绘

1. 增删可见dom

2. 修改元素位置或者大小时

3. 浏览器首次渲染时

4. 浏览器窗口大小发生改变时

5. 获取元素的一些属性时

优化

1. 不使用table布局

2. 避免多次设置内联样式,直接加类就好了

3. 开启一个图层,不用整个页面都重新渲染

4. 修改dom样式时,离线后再操作,只会导致一次回流

5. 不要再循环中频繁获取元素的宽高
